Hexo + Next 博客部署完整指南（含 GitHub Pages 常见问题排查）

Posted on 2025-10-19 | In Hexo , GitHub Pages

一．Hexo + Next 搭建个人博客为什么没有成功上传到自己的GitHub网站

1️⃣ 检查 Hexo 配置文件 _config.yml

Hexo 的部署主要依赖两个地方：

  • 根目录的 _config.yml
  • 主题目录下的 _config.yml（Next 主题有自己的配置）

# 根目录 _config.yml

  • url 一定要是你 GitHub Pages 的地址。
  • branch 默认 Hexo 是 gh-pages，但 GitHub 的默认分支可能是 main。
  • Next 主题 _config.yml 中 deploy 部分通常不需要改，主要关注根目录。
  • ！！！注意：用户名大小写不强制，但建议保持原样
  • ！！！注意：仓库名必须严格匹配 GitHub 上的大小写
  • ！！！注意：要更改根目录 _config.yml而非Next主题下的 _config.yml

2️⃣ 检查 GitHub 仓库和分支

  1. 确认你已经创建了仓库（例如 username.github.io）
  2. 分支是否匹配 Hexo 配置（main 或 gh-pages）
  3. 如果是 用户页面仓库（username.github.io），分支一般是 main
  4. 如果是 项目页面仓库（repo-name），分支一般是 gh-pages，并且 url 配置应加 /repo-name/

4️⃣ 检查 GitHub Pages 设置

  • 仓库 → Settings → Pages
  • 确认 Source 设置为你部署的分支（main 或 gh-pages）
  • 等待几分钟，GitHub Pages 才会生效

5️⃣ 常见错误

  1. 404 或网站不更新
    • 原因：url 或 root 配置错误
    • Next 主题 _config.yml 中 root 默认 /，如果是项目页面，需要改为 /repo-name/
  2. hexo deploy 报错
    • 原因：git 权限问题
    • 解决：确认 git remote add origin https://github.com/用户名/仓库.git 正确
    • 可以尝试使用 GitHub Personal Access Token 代替密码
  3. 主题资源加载失败
    • 原因：root 设置错误或主题静态文件路径错
    • Next 主题 _config.yml：
    • root: / # 用户页面
    • # 或
    • root: /repo-name/ # 项目页面

二．main和master分支有什么不同

1️⃣ 历史背景

  • 传统上，Git 初始化仓库的默认主分支叫 master。
  • 近年来，尤其是 GitHub 从 2020 年起，为了使用更中性的术语，把新建仓库的默认主分支改为 main。
  • 所以从 Git 功能上讲，master 和 main 没有本质区别，它们都是普通分支，只是默认名称不同。

2️⃣ 默认分支的作用

  • Git 仓库中的默认分支是：
    • 新建仓库时生成的首个分支
    • GitHub Pages 用户页面部署时默认读取的分支
  • 如果你创建的是 用户页面仓库（username.github.io），GitHub 默认使用 main 分支部署网站。
  • 如果你使用 项目页面仓库，你可以选择 gh-pages 或其他分支来部署。

✅ 总结

  • main 和 master 只是名字不同，本质相同。
  • 新建仓库默认是 main（GitHub），旧仓库是 master。
  • Hexo 部署必须保证 _config.yml 的 branch 与仓库默认分支一致。
  • 确保本地分支与远程分支一致
